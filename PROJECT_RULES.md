# 项目开发规则

本文档为接码团队管理平台的强制性开发约束，所有贡献者必须遵守。

## 代码规模与结构

1. **单个文件不得超过 300 行。**
2. **单个函数不得超过 60 行。**
3. **禁止把多个模块逻辑写在同一个文件。**
4. **后端必须模块化开发。**

## 职责边界

5. **前端只负责展示和交互，不允许写核心业务规则。**
6. **权限判断必须以后端为准。**

## 数据与状态

7. **所有余额变化必须走钱包流水表。**
8. **所有订单状态变化必须写操作日志。**
9. **所有供应商 API 必须通过 supplier adapter 调用。**
10. **所有数据库变更必须通过 migration。**
11. **禁止直接修改生产数据库。**
12. **所有状态必须使用枚举，不允许散落中文字符串。**

## 异步与开发流程

13. **所有后台任务必须放入 jobs，不允许在接口里面长时间阻塞。**
14. **新功能开发顺序必须是：数据库模型 → 后端接口 → 前端页面。**
15. **不允许为了赶进度写临时代码。**

## 架构与配置

16. **不允许一个页面直接调用多个无关模块。**
17. **不允许硬编码价格、供应商密钥、钱包地址、权限规则。**
18. **所有重要操作必须写 audit log。**
